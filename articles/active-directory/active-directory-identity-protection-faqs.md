# <a name="azure-active-directory-identity-protection-faq"></a>Ochrany identit Azure Active Directory – nejčastější dotazy


## <a name="why-do-some-risk-events-have-closed-system-status"></a>Proč některé události riziko mají stav "Uzavřeno (systém)"?

**Odpověď:** tyto jsou události, které Azure Active Directory Identity Protection zjištěny a později ukončeno, protože hello události byly nelze nadále považovat za rizikové. Tyto události není započítávat úroveň rizika hello uživatele.

## <a name="do-i-need-toobe-a-global-admin-toouse-identity-protection-in-hello-azure-portal"></a>Je nutné toobe toouse globálního správce ochrany identit v hello portál Azure?
**Odpověď:** **ne**. Může být buď čtečku zabezpečení, správce zabezpečení nebo globální správce toouse Identity Protection.
