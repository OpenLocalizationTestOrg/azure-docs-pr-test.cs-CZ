<span data-ttu-id="03d4c-126">Každý objekt, který obsahuje data, která je uložená ve službě Azure Storage (objekty BLOB, zprávy, entit a soubory) patří do oddílu, je identifikovaný klíč oddílu.</span><span class="sxs-lookup"><span data-stu-id="03d4c-126">Every object that holds data that is stored in Azure Storage (blobs, messages, entities, and files) belongs to a partition, and is identified by a partition key.</span></span> <span data-ttu-id="03d4c-127">Oddíl Určuje, jak vyrovnává zatížení Azure Storage objekty BLOB, zprávy, entit a soubory na serverech, aby vyhovovaly provozu těchto objektů.</span><span class="sxs-lookup"><span data-stu-id="03d4c-127">The partition determines how Azure Storage load balances blobs, messages, entities, and files across servers to meet the traffic needs of those objects.</span></span> <span data-ttu-id="03d4c-128">Klíč oddílu je jedinečný a je používaná k nalezení objektů blob, zprávy nebo entity.</span><span class="sxs-lookup"><span data-stu-id="03d4c-128">The partition key is unique and is used to locate a blob, message, or entity.</span></span>

<span data-ttu-id="03d4c-129">V tabulce výše uvedeném v [cíle škálovatelnosti pro standardní účty úložiště](#standard-storage-accounts) uvádí výkonnostní cíle pro jeden oddíl pro každou službu.</span><span class="sxs-lookup"><span data-stu-id="03d4c-129">The table shown above in [Scalability Targets for Standard Storage Accounts](#standard-storage-accounts) lists the performance targets for a single partition for each service.</span></span>

<span data-ttu-id="03d4c-130">Oddíly ovlivnit Vyrovnávání zatížení a škálovatelnost pro jednotlivé služby úložiště následujícím způsobem:</span><span class="sxs-lookup"><span data-stu-id="03d4c-130">Partitions affect load balancing and scalability for each of the storage services in the following ways:</span></span>

* <span data-ttu-id="03d4c-131">**Objekty BLOB**: klíč oddílu pro objekt blob je název účtu + název kontejneru + název objektu blob.</span><span class="sxs-lookup"><span data-stu-id="03d4c-131">**Blobs**: The partition key for a blob is account name + container name + blob name.</span></span> <span data-ttu-id="03d4c-132">To znamená, že každý objekt blob může mít svůj vlastní oddíl Pokud zatížení u objektu blob se vyžaduje.</span><span class="sxs-lookup"><span data-stu-id="03d4c-132">This means that each blob can have its own partition if load on the blob demands it.</span></span> <span data-ttu-id="03d4c-133">Objekty BLOB mohou být distribuovány na počtu serverů, aby bylo možné škálovat přístup k nim, ale jediného objektu blob může obsluhovat pouze jeden server.</span><span class="sxs-lookup"><span data-stu-id="03d4c-133">Blobs can be distributed across many servers in order to scale out access to them, but a single blob can only be served by a single server.</span></span> <span data-ttu-id="03d4c-134">Zatímco objekty BLOB mohou být logicky seskupeny do kontejnery objektů blob, neexistují žádné rozdělení dopady z toto seskupení.</span><span class="sxs-lookup"><span data-stu-id="03d4c-134">While blobs can be logically grouped in blob containers, there are no partitioning implications from this grouping.</span></span>

* <span data-ttu-id="03d4c-139">**Entity**: klíč oddílu pro entitu je účet název + název tabulky + oddílu klíč, kde klíč oddílu je hodnota požadované uživatelem definované **PartitionKey** vlastnost pro entitu.</span><span class="sxs-lookup"><span data-stu-id="03d4c-139">**Entities**: The partition key for an entity is account name + table name + partition key, where the partition key is the value of the required user-defined **PartitionKey** property for the entity.</span></span> <span data-ttu-id="03d4c-140">Všechny entity se stejnou hodnotou klíče oddílu jsou seskupené do stejného oddílu a zpracovává stejný server oddílu.</span><span class="sxs-lookup"><span data-stu-id="03d4c-140">All entities with the same partition key value are grouped into the same partition and are served by the same partition server.</span></span> <span data-ttu-id="03d4c-141">Toto je bod důležité pochopit, při navrhování vaší aplikace.</span><span class="sxs-lookup"><span data-stu-id="03d4c-141">This is an important point to understand in designing your application.</span></span> <span data-ttu-id="03d4c-142">Aplikace by měla vyvážit výhody škálovatelnost šíření entity napříč více oddílů s výhody přístup dat seskupování entity v jeden oddíl.</span><span class="sxs-lookup"><span data-stu-id="03d4c-142">Your application should balance the scalability benefits of spreading entities across multiple partitions with the data access advantages of grouping entities in a single partition.</span></span>  

<span data-ttu-id="03d4c-143">Klíčové výhody na seskupení sady entit v tabulce do jednoho oddílu je, aby bylo možné provádět atomic dávkových operací na entity v stejného oddílu, protože oddíl existuje na jednom serveru.</span><span class="sxs-lookup"><span data-stu-id="03d4c-143">A key advantage to grouping a set of entities in a table into a single partition is that it's possible to perform atomic batch operations across entities in the same partition, since a partition exists on a single server.</span></span> <span data-ttu-id="03d4c-144">Pokud chcete provést dávkové operace pro skupinu entit, zvažte proto je seskupit se stejným klíčem oddílu.</span><span class="sxs-lookup"><span data-stu-id="03d4c-144">Therefore, if you wish to perform batch operations on a group of entities, consider grouping them with the same partition key.</span></span> 

<span data-ttu-id="03d4c-145">Na druhé straně entit, které jsou ve stejné tabulce, tabulka však mají různé klíče oddílů může být zatížení rovnoměrně rozdělen mezi různými servery, které umožňují mít větší škálovatelnost.</span><span class="sxs-lookup"><span data-stu-id="03d4c-145">On the other hand, entities that are in the same table but have different partition keys can be load balanced across different servers, making it possible to have greater scalability.</span></span>
