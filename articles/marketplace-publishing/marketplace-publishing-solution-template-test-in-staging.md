# <a name="test-your-solution-template-offer-in-staging"></a>Otestovat vaši nabídku šablony řešení v pracovní
Pracovní znamená nasazení vaši nabídku v soukromé "izolovaného", kde můžete otestovat a ověřit jeho funkčnost před odesláním do produkčního prostředí. Nabídka se zobrazí v pracovním stejně, jako by zákazníkovi, který je nasazena. Chcete-li být vložena do přípravy, musí mít certifikovanou vaši nabídku.

Po nabídka je připravený, můžete zobrazit a otestovat nabídku v [portálu Azure](https://portal.azure.com/).

Použijte následující postup push vaši nabídku pro pracovní a otestovat ho [portálu Azure](https://portal.azure.com/):

1. Přejděte na [publikování portál](https://publish.windowsazure.com) > **šablony řešení** kartě > vaši nabídku > **publikovat** > **nabízená pracovní**.
2. Zadejte seznam předplatných Azure, které budete používat pro zobrazení náhledu a otestovat vaši nabídku.
3. Přihlaste se k portálu Azure preview pomocí ID odběru, který jste použili v předchozím kroku.
4. Provádění alespoň jeden zaokrouhlit testování na portálu Azure preview na bodech uvedených níže:
   * Ujistěte se, že marketingové obsah se zobrazí správně v Azure Marketplace.
   * Topologie nasazení začátku do konce.
   * Proveďte test výkonnosti a zátěžové testování.
   * Ujistěte se, že vaše topologie dodržuje osvědčené postupy.

## <a name="next-steps"></a>Další kroky
Pokud jste s výsledky spokojeni, pak můžete přejít k publikování fázi konečné nabídka **krok 4**: [nasazení vaši nabídku Marketplace](marketplace-publishing-push-to-production.md). Jinak proveďte změny v vaši nabídku a požádat o certifikaci znovu.
